# 00. ロードマップ

サービス開始までの全体マイルストーンを管理するドキュメントである。詳細な仕様は各番号のドキュメントに委ね、ここでは「いつ・何を・どの順で」を俯瞰する。

## 全体像（5 ステップ）

```
Step 1 企画の言語化      →  01-product-requirements.md
Step 2 パーツデータ準備   →  02-parts-data.md
Step 3 デザイン設計       →  03-design-system.md
Step 4 開発・デプロイ     →  04-architecture.md / 05-development-plan.md
Step 5 テスト・ローンチ   →  06-launch-plan.md
```

個人開発における最速リリースを最優先とし、「仕様の引き算」で MVP に集中する。

## マイルストーン

| フェーズ | ゴール | 主な成果物 | 完了条件 |
|---|---|---|---|
| M0: 企画確定 | スコープの確定 | PRD（01） | MVP 機能一覧が確定している |
| M1: データ準備 | 初期マスタ整備 | パーツ CSV/Seed（02） | 主要パーツ 100〜200 件を投入済み |
| M2: デザイン確定 | UI 方針とOGP設計 | デザインシステム（03） | カラー・カード UI・OGP レイアウトが確定 |
| M3: 基盤構築 | DB・認証の稼働 | スキーマ / 認証（04） | ユーザー登録・ログインが動作 |
| M4: コア機能実装 | セッティング作成〜共有 | 実装（05） | 選択→合算→保存→公開 URL が通る |
| M5: 動的 OGP 実装 | シェア画像の生成 | OGP 実装（03/05） | SNS 貼付でカード画像が表示される |
| M6: クローズドテスト | バグ・UX 改善 | フィードバック（06） | 主要導線に致命バグなし |
| M7: 初期ローンチ | コミュニティ公開 | ローンチ投稿（06） | X で公開・初期投稿が蓄積 |

## クリティカルパス

最速リリースの観点で、以下が並行不可の直列依存となる。

1. **PRD 確定（M0）** が全ての前提。ここを曖昧にすると後工程が揺れる。
2. **DB スキーマ（M3）** が固まらないとコア機能（M4）に着手できない。
3. **動的 OGP（M5）** は本サービスの CGM マーケティングの心臓部。MVP に含めることを強く推奨する。

## スコープ管理の原則

- 迷ったら「やらない」を選ぶ（引き算の徹底）。
- スコープ外機能（レビュー・コメント・EC 導線）は Issue 化して将来バックログに積む。
- 詳細は [01. PRD](./01-product-requirements.md) の「スコープ外」を参照。
